Right: KEY INSIGHTS (button)

  01  "The product had depth but no surface."
      Three taxonomies — Categories, Labels, Tags — that even the founder stumbled over mid-sentence. When the person who built it can't explain it cleanly, a new user has no chance.
      Tag: IA finding

  02  "The timeline punished real work."
      Planned time blocks treated any deviation as failure. But the founder's own data showed fragmented, interrupted work was the norm — not the exception. The interface was telling a lie about how work happens.
      Tag: UX finding

  03  "The AI was powerful and hidden."
      Task summaries, conversational agent, voice transcription, screenshot analysis — all working. But the configuration screen was a wall of 20+ checkboxes with no presets and no explanations. The secondary brain had a locked door.
      Tag: Pain point

Bottom green bar:
  The insight that drove everything: the product had depth. It just had no surface.

MODULE 13 — WHAT I'D RETHINK
───────────────────────────────────────
Layout: Clean, reflective. Dark bg, white text.

WHAT I'D RETHINK (label)

  Sample size of one.
  Matthias had 3,000+ tasks of personal usage data — invaluable for understanding the product's behavior. But he was also the only user. He had learned to work around interface issues that would block a new user cold. Even three external beta testers during the audit phase would have surfaced patterns the founder's muscle memory had absorbed.

  Prototype the hard UI earlier.
  The Health Score comparison was effective but came late. If I'd prototyped it during Milestone 1, the decision would have been baked into the Milestone 2 timeline work rather than retrofitted. When you know a UI problem has multiple valid answers, don't wait to test them.

Bottom pull quote:
  "Matthias built a secondary brain that only he could use. The design work wasn't about making it prettier. It was about making it work for everyone else."
